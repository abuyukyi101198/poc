Datacenter Network Topology

Option C  ·  Standard Leaf-Spine Pod  ·  Radial Layered (Sugiyama) Dendrogram Graph

About this Visualization

This chart shows a datacenter network as concentric rings (Disc View) or, for large multi-pod deployments, as a mirrored layered chart (Stack View). Both views draw the same underlying topology — a set of routing/containment hierarchies that all share physical racks and servers.

Why three chart techniques, combined?

  • Sugiyama-style layered layout — each ring (Pod, Spine, Leaf, Rack, Server…) is one layer. Nodes within a ring are ordered to minimize link crossings with the ring next to it, the same goal as a classic layered/hierarchical graph drawing.
  • Radial dendrogram geometry — instead of straight horizontal bands, layers are bent into concentric circles. Every ring keeps a full 360° of arc regardless of depth, so the outermost ring (Server) — which has the most nodes — gets the most space, and links read as smooth radial-then-circular "tree of life" paths rather than straight diagonal lines.
  • Icicle-chart containment (Stack View only) — at multi-pod scale, giving every pod its own angular wedge would leave too little room per pod. Stack View switches to a partition/icicle layout instead: each pod is a column, and a parent's box width is simply the union of its children's widths — the same rule used by flame graphs. This scales to many pods far better than a single disc.

In short: Sugiyama decides ring order/crossing-minimization, the radial dendrogram decides how a single pod is drawn in detail, and the icicle chart takes over for the zoomed-out, many-pod overview. Clicking a Stack View column drills into that pod's full Disc View.

Reading the chart

  • Rings, inner → outer: Pod → Superspine → Spine → Leaf → Rack → Server (some rings are absent at smaller scales — see the Option tabs).
  • Arc width = relative size (server count) of that node's subtree.
  • Orange = Data plane link.
  • Purple = Management plane link.
  • Grey = physical containment link (rack → server).
  • Gold = peer link (e.g. ToR-to-ToR).
  • Sage green = Border Leaf (external/WAN connectivity point).

Interactions

  • Hover a node — highlights its full ancestor lineage (all parents up to the Pod) and dims everything unrelated. Multiple parents may light up at once, since a rack or leaf can have more than one upstream device.
  • Hover any node — shows a detail tooltip (weight, plane, trust tier, AZ, MAAS controller IDs, etc.) — only sections with data are shown.
  • Plane toggle buttons — show/hide Data, Management, Containment, or Peer Adjacency links independently. Ring geometry never changes, only link visibility.
  • Option tabs (A–D) — switch between reference deployment sizes, from a single switch/rack to a full multi-pod fabric.
  • Stack View column click (Option D only) — drills Disc View into that pod alone.
  • Theme toggle (☀/☾) — switches dark/light color scheme; your choice is remembered.
